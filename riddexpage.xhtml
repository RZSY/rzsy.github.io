RizqXP Video Player

RizqXP Video Player is my first try at trying to separate the functions. It was quite revolutionary, though not much different, both problems from Riddex Gen 1 was still very apparent. However, RizqXP Video Player, or Riddex Gen 2 paved a way for the next and current iteration of the Riddex Video Player family. It shows that interoperability between different .js (JavaScript) files is possible and separating functions in different script files would streamline (pardon the jive) the task of making the player reusable and not a one time thing.

Riddex Video Engine (Riddex VE)

Riddex Video Engine is not perfect. However, it has the ability to enter fullscreen, and is interoperable through a simple-er script line. That of course links to a custom file that houses the JavaScript that allows it to function (riddexscript.js). It is not VLC media player, it is not GOM, it doesn't have many fancy features, but it is stable. Riddex Video Engine was born because of my personal hatred towards the object tag. The object tag is my enemy, because of its unreliability, I had to spend so many hours working on an engine.

Riddex Video Engine as shown above, can be reused repeatedly without too much changes.

Riddex Player and its modular approach is operated on the .xhtml page through two modules. Module A, and Module B. Module A is where the Video player be placed and Module B is the defining module. However, this is where a con rises up. One page can only have 1 playing video file (You can add the same video a hundred times if you want, but it currently doesn't support any video exchange). So if you have 20 videos, you would need 20 pages. This is a serious weakness of the Riddex Video Engine, however, for an early development engine, it is currently working way better when compared to its predecessors Riddex Gen 1 and RizqXP Video Player.

These two allows the video to be loaded in the page properly. It is rather tedious having two different modules at the same time. But it is pretty stable so far. I'm not sure if the stability is held together with concrete or with hopes and dreams of a pitiful child, what I do know is that it works and as some of the programmers I know say "Don't fix what's not broken,"

Riddex VE Pros Cons
Uses CDATA Wrapper. Easy to implement in already existing .xhtml files. Could cause problems with certain browsers.
Uses 2 (two) separate modules in one .xhtml file. Easy to implement in already existing .xhtml files. Can only have one video file playing per page.
Has its own script page. Easy to implement in already existing .xhtml files. Might conflict with another script file if the .xhtml file uses more than 1 (one) script files
